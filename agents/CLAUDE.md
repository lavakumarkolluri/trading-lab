# agents/ — Report Output Directory

This directory holds **output files** generated by the three Python pipeline agents. Agent source code lives in `pipeline/`, not here. Claude Code agents are configured globally at `~/.claude/agents/` — unrelated to this directory.

## Active Subdirectory

- `agents/analysis/` — weekly Markdown trade reports written by `analysis_agent.py`
  - Format: `YYYY-MM-DD.md` (one file per Sunday run)
  - Controlled by `REPORT_DIR=/trading-lab/agents/analysis` env var on the `analysis_agent` container

## Reserved Subdirectories (currently empty)

`data/`, `journal/`, `learning/`, `loader/`, `monitor/`, `signal/` — reserved for future pipeline agent output. Do not create files here manually.

---

## The Three Pipeline Agents

All agent source code is in `pipeline/`. Do not add Python files to this directory.

| Agent | Source | Schedule | Purpose |
|---|---|---|---|
| `signal_agent` | `pipeline/signal_agent.py` | Mon–Fri 13:10 UTC | Explains today's confidence scores via Telegram — feature-by-feature breakdown of why each symbol scored high/low |
| `analysis_agent` | `pipeline/analysis_agent.py` | Daily 17:15 UTC + weekly Sunday 08:30 UTC | Daily trade P&L summary via Telegram + full weekly Markdown report written to `agents/analysis/` |
| `monitor_agent` | `pipeline/monitor_agent.py` | Always-on (5-min poll, 09:00–16:00 IST) | Real-time position alerts: entry detected, target hit, stop loss, trailing stop activated, EOD warning, position closed |

### Docker Compose Behaviour

- `monitor_agent` — `restart: unless-stopped` (always running during market hours)
- `signal_agent`, `analysis_agent` — `restart: "no"` (scheduler triggers each run via `docker compose run`)

### Telegram Configuration

All three agents use `TELEGRAM_BOT_TOKEN` and `TELEGRAM_CHAT_ID` env vars. If either is unset, the agent logs a mock alert to stdout instead of sending — safe for local dev.

### Key Tables

| Agent | Reads | Writes |
|---|---|---|
| `signal_agent` | `analysis.confidence_scores` | `system_meta.alert_log` |
| `analysis_agent` | `trades.trade_outcomes` | `system_meta.alert_log`, `agents/analysis/*.md` |
| `monitor_agent` | `trades.open_positions`, `market.options_chain` | `system_meta.alert_log` |
